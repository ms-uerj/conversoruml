Utilizando JDOM para ler e gerar XML

Se você já usou XML em alguma aplicação Java e utilizou das APIs DOM ou SAX então com certeza você já se deparou com a complexidade delas. Foi pensando nisso que Jason Hunter e Brett McLaughlin criaram essa fantástica e tão simples API denominada JDOM.

Antes de iniciarmos é importante lembrar que o JDOM não é um parser, como o Xerces ou o JAXP, ele é uma API que facilita a criação e atualização de documentos XML. Por definição ele é configurado para utilizar o JAXP mais pode ser configurado para usar a maioria dos parsers existentes.

Nesse tutorial estarei explicando como criar e como ler documentos XML e no próximo veremos conceitos mais avançados como validação e atualização, utilizaremos como um exemplo prático um cadastro de mensagens. Vale lembrar que em primeiro lugar devemos definir bem a estrutura dos documentos XML, mais isso não será visto aqui, uma vez que não é o objetivo deste tutorial.

Consideramos a seguinte estrutura:
  1. <mural>  
  2.    <mensagem id="" prioridade="">    
  3.       <para></para>  
  4.       <de></de>  
  5.       <corpo></corpo>  
  6.    </mensagem>  
  7. </mural>  


O exemplo acima podeira ser utilizado para a criação de um sistema de mensagens ou um mural onde as pessoas vão inserindo suas mensagens que ficarão disponíveis para os outros usuários, ou também dentro de um organização para facilitar a troca de informações entre os funcionários.

1- Criando um documento XML.

Antes de criarmos um documento, vamos conhecer um pouco da API JDOM que representa um documento XML bem como todas as suas propriedades:

Document: Esta classe representa um documento xml inteiro, esta pode conter apenas um elemento que é denominado root, comentários e outros elementos de definição.

Element: Esta classe representa um elemento XML, um elemento XML também conhecido como nó é pode conter outros elementos, atributos e comentários além é claro de um valor.

Attribute: Esta classe representa um atributo que pode ser atribuido a um elemento.

Comment: Esta classe representa um comentário em XML, os comentários em XML são representados com a seguinte sintaxe
.

À medida em que vamos percorrendo o tutorial, vamos conhecendo cada vez mais a API JDOM. A documentação completa pode ser vista através do link no fim deste tutorial.

Vamos ao código.
  1.     //Declaração dos elementos que irão compor a estrutura do documento.  
  2.    Element mural = new Element("mural");  
  3.    Element mensagem = new Element("mensagem");  
  4.    Element para = new Element("para");  
  5.    Element de = new Element("de");  
  6.    Element corpo = new Element("corpo");  
  7.   
  8.    //"Setando" os atributos  
  9.    mensagem.setAttribute("id", "01");  
  10.      
  11.    //"Setando" outro atributo agora utilizando da classe Attribute  
  12.    Attribute prioridade = new Attribute("prioridade","-1");  
  13.    mensagem.setAttribute(prioridade);     
  14.   
  15.    mensagem.addContent(para);  
  16.    mensagem.addContent(de);  
  17.    mensagem.addContent(corpo);  
  18.      
  19.    mural.addContent(mensagem);  
  20.         
  21.    //Criando o documento XML (montado)  
  22.    Document doc = new Document();  
  23.    doc.setRootElement(mural);  
  24.   
  25.    //Imptrimindo o XML  
  26.    XMLOutputter xout = new XMLOutputter("  ", true);  
  27.    xout.output(doc, System.out);  


Da linha 1 à 6 são declarados os elementos que irão compor o corpo do documento, repare que os nomes dados aos objetos são iguais ao nome do elemento que cada um corresponde, isso é para facilitar no desenvolvimento (e também é uma boa prática de programação).

A linha 8 faz a inserção do atributo id no elemento mensagem através de uma das declarações do método setAttribute. As linhas 11 e 12 fazem a inserção de um segundo atributo no elemento mensagem, desta vez, utilizando de um objeto da classe Attribute. O método setAttribute ainda possui ainda duas outras variações a primeira que recebe o nome, o valor e o namespace¹ do atributo e a outra que recebe um objeto do tipo java.util.List.

Quote:
¹ Apesar da API JDOM dar suporte completo a namespaces, esse conceito não será visto nesse tutorial.



Nas linhas 14 à 16 adicionamos ao elemento mensagem, os elementos(children) que compõem o seu corpo.

O elemento root segundo o documento que definimos acima, é o elemento mural. Este por sua vez possui outros elementos que estão sendo adicionados na linha 18. Repare que o elemento mural é apenas um elemento comum até agora, oque mostra a facilidade de criamos a estrutura do documento, somente adicionando filhos à um elemento pai.

Na linha 21 estamos declarando e inicializando um documento, este documento é apenas uma declaração de documento vazia, uma

vez que ainda não possui um elemento root, na linha 22 estamos informando que o elemento "mural" será o elemento root para este documento.

Finalizando, na linha 25 e 26 criamos um objeto XMLOutputter e mandamos escrever o documento na saida padrão², repare na flexibilidade que a classe XMLOutputter nos oferece, podemos "escrever" com ela qualquer elemento, comentário atributo em qualquer saída OutputStream, falaremos mais sobre a classe XMLOutputter durante o decorrer do tutorial (próxima parte, JDOM Intermediário).

Quote:
Agora que você já sabe como escrever documentos XML, que tal reescrever o código acima para gerar seus arquivos?

2- Lendo arquivos XML

Agora que já conhecemos as classes que compõem a estrutura de um documento ficará mais fácil para lermos os arquivos, uma vez que o processo é o contrário da escrita e isso se torna mais evidente ainda na utilização da API JDOM. Agora apresentarei a vocês uma classe que será vossa parceira por muitos programas, ela é reponsável pela leitura dos arquivos XML.

SAXBuilder: é utilizada para construir a árvore XML à partir de um arqivo ou streams utilizando SAX. Outra classe de entrada muito importante é a classe DOMBuilder, que executa a mesma tarefa que a SAXBuilder, mas é mais lenta pois antes de disponibilizar os valores ela precisa carregar toda a árvore XML ns memória. Para esse tutorial iremos utilizar a classe SAXBuilder, mais você pode fazer experimentos com a DOMBuilder apenas substituindo as classes.

Consideramos que tenhamos um arquivo no diretório raiz chamado mural.xml

Vamos ao código:

  1.    //Aqui você informa o nome do arquivo XML.  
  2.    File f = new File("c:/mural.xml");  
  3.   
  4.    //Criamos uma classe SAXBuilder que vai processar o XML4  
  5.    SAXBuilder sb = new SAXBuilder();  
  6.   
  7.    //Este documento agora possui toda a estrutura do arquivo.  
  8.    Document d = sb.build(f);  
  9.   
  10.    //Recuperamos o elemento root  
  11.    Element mural = d.getRootElement();  
  12.   
  13.    //Recuperamos os elementos filhos (children)  
  14.    List elements = mural.getChildren();  
  15.    Iterator i = elements.iterator();  
  16.   
  17.    //Iteramos com os elementos filhos, e filhos do dos filhos  
  18.    while (i.hasNext()) {  
  19.       Element element = (Element) i.next();  
  20.       System.out.println("Códido:"+ element.getAttributeValue("id"));  
  21.       System.out.println("Prioridade:"+ element.getAttributeValue("prioridade"));  
  22.       System.out.println("Para:"+ element.getChildText("para"));  
  23.       System.out.println("De:"+ element.getChildText("de"));  
  24.       System.out.println("Corpo:"+ element.getChildText("corpo"));  
  25.    }  


Na linha 1 definimos o arquivo XML que iremos processar. Em seguida na linha 4 criamos uma classe SAXBuilder que será usada para processar o arquivo.

O método build(File) que possui ainda outras assinaturas, cria um documento (Document) que irá conter a estrutura do arquivo que definimos. Após isso, recuperamos o elemento root, linha 10, e em seguinda todos os elementos filhos do elemento root, linha 13.

Na iteração dos elementos, linha 17, utilizamos de métodos "get" para pegarmos os valores de conteúdo e dos atributos, como pode ser visto nas linhas 19 à 23.

Já está em fase de desenvolvimento um novo tutorial que tratará de aspéctos mais avançados do JDOM e terá como pré-requisito a leitura deste.
